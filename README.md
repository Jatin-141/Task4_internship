# Task 4 – Interactive Dashboard Design

## Objective

To design an interactive Power BI dashboard for business stakeholders that enables data-driven decision making through KPIs, trend analysis, and dynamic filtering.

---

## Tools Used

- Power BI Desktop
- Sales Financial Dataset (SuperStore)
- DAX for KPI calculations

---

## Dashboard Structure

### Page 1 – Executive Overview

This page provides a high-level summary of business performance.

Included components:

- **Total Sales KPI**
- **Total Profit KPI**
- **Total Quantity Sold**
- **Sales Growth % (Year-over-Year)**
- **Sales Trend Over Time**
- **Profit by Sub-Category**
- **Sales by Region**
- **Region Slicer**
- **Date (Year) Slicer**

Purpose:
To provide stakeholders with quick performance insights and allow dynamic filtering.

---

### Page 2 – Detailed Analysis

This page focuses on deeper performance breakdowns.

Included components:

- Expanded Profit by Sub-Category analysis
- Regional sales comparison
- Year-wise sales trend
- Interactive Region and Date slicers

Purpose:
To enable drill-down analysis for identifying regional and product-level performance patterns.

---

## Key KPIs Implemented

- Total Revenue
- Total Profit
- Total Units Sold
- Sales Growth % (DAX Time Intelligence Measure)

---

## Interactivity Features

- Dynamic filtering using slicers (Region & Year)
- KPI values update automatically based on filters
- Trend analysis responsive to time selection
- Growth calculation adjusts based on selected region/year

---

## Business Insights

- Revenue and profit trends over time.
- Regional sales distribution comparison.
- Identification of profitable and low-performing sub-categories.
- Year-over-year performance evaluation.

---

## Outcome

This dashboard demonstrates the ability to:

- Design stakeholder-focused dashboards
- Apply DAX for KPI and growth calculations
- Implement interactive filtering
- Maintain consistent visual hierarchy and theme
- Structure multi-page analytical dashboards
